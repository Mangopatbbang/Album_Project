## 페이지별 설계

### 1. 홈 (`/`)

**레이아웃 변경:**

```
[데스크탑 — 2컬럼]
┌──────────────────────┬──────────────────────────────────┐
│  오늘의 앨범 (큰 카드) │  최근 활동 피드                    │
│  제목 + 아티스트       │  (rating / comment / playlist)   │
│  평균 점수 + 바         │  아바타 + 내용 + 타입 아이콘 + 시간│
│  소감 2-3줄 미리보기   │                                  │
│  [앨범 열기] [공유]    │                                  │
├──────────────────────┴──────────────────────────────────┤
│  청음집 목록 (가로 스크롤 카드)                            │
│  커버 콜라주 + 제목 · 작성자 · N장                         │
└─────────────────────────────────────────────────────────┘

[모바일]
오늘의 앨범 (커버 좌측 정사각형 + 우측 텍스트)
최근 활동 피드 (세로 스택)
청음집 목록 (가로 스크롤)
```

**변경 포인트:**
- 피드 항목 타입 아이콘: ★ 평점 / " 소감 / ♪ 청음집
- 피드에 liked_tracks 표시: "♥ 3곡 찜"
- 오늘의 앨범 카드에 "공유" 버튼 → 앨범 전용 페이지 링크 복사
- 청음집 섹션: 최신 3개 + [전체 보기] → `/playlists`

---

### 2. 음반고 (`/albums`)

**변경 포인트:**
- 정렬 옵션 추가: 최신 등록 | 평점 높은순 | 아티스트명 | 연도
- 장르 chip에 개수 뱃지: `힙합 (12)`
- AlbumCard에 "아직 안 들은" 뱃지 (로그인 유저 + 미평가 앨범)
- 앨범 클릭 → Option C 인터셉팅 라우트 모달, URL `/album/[id]`로 변경

---

### 3. 앨범 전용 페이지 `/album/[id]` — 신규

**Option C 구현:**
- 음반고 클릭 → 인터셉팅 라우트 모달 (현재 AlbumModal 재사용)
- URL이 `/album/[id]`로 push됨 → 공유 가능
- 직접 접근 시 전용 페이지 렌더링

```
[데스크탑 — 2컬럼]
┌────────────────────────┬──────────────────────────────┐
│  커버 이미지 (정사각형)  │  제목 / 아티스트 / 연도 / 장르  │
│  Spotify 링크           │  평균 점수 + 점수 분포          │
│                        │  [소감 쓰기] [평점 주기]        │
│                        │  [링크 복사] 공유 버튼           │
├────────────────────────┴──────────────────────────────┤
│  트랙리스트 (펼치기/접기)  │  멤버 평점 목록               │
├───────────────────────────────────────────────────────┤
│  extra_artists · 태그                                 │
└───────────────────────────────────────────────────────┘

[모바일]
커버 전폭 (흐림 배경 + 선명 커버 fade-in)
제목 / 아티스트 / 점수
트랙리스트 (기본 접힌 상태)
멤버 평점 카드
소감 목록
```

**추가 사항:**
- SEO 메타태그 (`og:image`, `og:title`)
- 뒤로가기 버튼

---

### 4. 청음감 (`/reviews`) — 소감 피드

**변경 포인트:**
- liked_tracks 표시: "♥ 3. 곡명, 7. 곡명"
- 소감 전문 검색 추가
- 소감 카드에서 앨범 클릭 → Option C 모달

---

### 5. 청음평 (`/ratings`) — 평점 피드

**변경 포인트:**
- rating_history 초기 평점 기록 버그 수정 (현재 `prevScore !== score`일 때만 기록 → 최초 등록도 기록)
- 점수 변화량 표시: `8.0 → 9.0 (+1.0)` 또는 `첫 청음`
- 재청음 이벤트: 기존 평점 앨범 재청음 기록 버튼

---

### 6. 청음인 (`/users`)

**변경 포인트:**
- 멤버 카드에 최근 청음 앨범 커버 미니 그리드 (3개)
- InsightSection 대표값 노출: "공동체와 가장 다른 눈"

---

### 7. 프로필 (`/profile/[userId]`)

**현재 순서:**
헤더 → 명반전 → 아티스트 → 최근감상 → ... → InsightSection (맨 아래)

**변경 순서:**
```
헤더 (아바타 + 이름 + 소개글 + 통계)
InsightSection ← 최상단 이동 (핵심 개성 지표)
최근 청음 (6장 그리드)
청음집 목록
소감 목록 (최근 5개 + 더보기)
```

```
[데스크탑 — 2컬럼]
┌──────────────────────────┬───────────────────────────────┐
│  아바타 80px + 이름        │  청음 N장 · 평균 N점 · 소감 N개 │
│  소개글 (편집 가능)        │  장르 분포 chip                │
└──────────────────────────┴───────────────────────────────┘
이견 앨범 · 숨은 명반 (InsightSection)
최근 청음 그리드
청음집 목록
소감 목록
```

**추가 사항:**
- 소개글 `bio` 필드 (profiles 테이블에 컬럼 추가 필요)
- 청음집 목록 프로필에 통합

### 8. 청음집 목록 `/playlists` — 신규 페이지

```
[데스크탑]
[+ 새 청음집] 우상단
카드 그리드 (3열):
  커버 콜라주 2x2
  제목 / 작성자 / N장 / 날짜

[모바일] — 2열
```

**청음집 강화:**
- 이름: "선곡집" → "청음집" (UI 텍스트만, DB 컬럼명 유지)
- 생성 시 제목 직접 입력 (현재 날짜 자동생성 → 직접 입력으로)
- 공개/비공개 설정 (기본 공개, `is_public` 컬럼 추가 필요)
- 공유 버튼 `/playlist/[id]` 페이지에 추가

### 9. 알림 — 모바일 진입점 추가

**현재:**
- Header.tsx에 전체 알림 시스템 구현 완료 (bell + unread badge + dropdown)
- `hidden sm:block` — 모바일에서 완전히 숨겨짐
- BottomNav에 진입점 없음

**변경:**
- BottomNav에 🔔 탭 추가 (미읽음 뱃지 포함)
- 기존 6탭 중 사용 빈도 낮은 항목 정리 또는 7탭으로 확장
- 데스크탑 Header 알림은 현재 그대로 유지

### 10. 검색 강화

**현재:** 앨범 제목 + 아티스트 텍스트 매칭  
**추가:**
- 소감 전문 검색 (`comment` 필드 full-text search)
- 검색 결과 탭 분리: 앨범 | 소감 | 청음집 | 멤버

## 구현 순서

### Phase 1 — 프론트엔드만 (DB 변경 없음)
1. InsightSection 프로필 최상단 이동
2. 청음집 이름 "선곡집" → "청음집" (텍스트 교체)
3. 알림 — BottomNav 모바일 진입점 추가
4. 피드 항목에 liked_tracks 표시
5. 빈 상태 안내 카드
6. 청음집 생성 시 제목 직접 입력
7. 음반고 정렬 옵션 추가

### Phase 2 — 라우팅 변경
8. 앨범 전용 페이지 `/album/[id]` 신설 (Option C + 인터셉팅 라우트)
9. `/playlists` 청음집 전체 목록 페이지 신설
10. 프로필에 청음집 목록 통합

### Phase 3 — DB 변경 포함
11. rating_history 초기 평점 기록 버그 수정
12. 소개글 `bio` 필드 추가 (profiles 테이블)
13. 청음집 `is_public` 컬럼 추가
14. 소감 전문 검색 (DB 인덱스)
15. 재청음 이벤트 기록
16. 온보딩 팝업 (첫 로그인 플래그)

## 마일스톤

### 2025-12-01 — 첫 커밋
구글 시트 + Next.js 조합으로 가장 원시적인 형태의 앨범 기록 앱 시작.  
멤버별 평점이 앨범 테이블의 개별 컬럼이었다. (4개 컬럼 고정)

### 2026-01-14 — DB 전환
구글 시트 의존 완전 제거. Supabase로 완전 이전.  
처음으로 API 레이어(`/api/albums`) 구축.

### 2026-03-28 — 앱 전면 재구축
단일 페이지에서 멀티페이지 앱으로 전환.  
홈 / 음반고 / 청음감 / 청음집 / 청음인 / 프로필 구조 확립.  
Supabase Auth 로그인 도입 — 처음으로 "멤버십" 개념 생김.

### 2026-03-31 — 핵심 기능 완성
- 트랙 좋아요
- 프로필 편집 (아바타 업로드)
- Watchlist (나중에 듣기)
- 무한 스크롤
- 7·8점 Glow 효과

이 시점부터 내부 서비스로서 일상적으로 사용 가능한 수준이 됨.

### 2026-04-08 — Spotify로 완전 전환
iTunes 검색을 버리고 Spotify 검색으로 전환.  
트랙리스트 + 커버 + 메타데이터를 단일 API에서 통합 수집.

### 2026-04-12 — 커뮤니티 기능 완성
아티스트 모달 (디스코그라피), 평 컨베이어벨트, 프로필 리디자인.  
"앨범 기록 앱"에서 "청음 커뮤니티"로 성격이 확장됨.

### 2026-05-16 — v2.0 대규모 리디자인
홈 / 청음감 / 프로필 전면 개편.  
연간 리캡, Pearson 취향 궁합, 논란 지수 등 분석 기능 추가.  
View Transitions 적용 — 처음으로 앱처럼 느껴지는 전환 효과.

### 2026-05-18 — FilterSelect & 프로필 사진
네이티브 `<select>` 전면 교체 → 커스텀 FilterSelect (바텀시트 방식).  
이모지 → 실제 프로필 사진(UserAvatar) 도입.  
자체 이벤트 트래킹 시스템 구축 (Vercel Analytics 유료 전환 대응).

### 2026-05-19 — 어드민 데이터 탭 & 스케일 대응
어드민에 사용자 활동 대시보드 추가.  
테스터 수 증가를 대비한 성능 최적화 작업 시작.

### 2026-05-20 — 트랙 재생시간 + 퍼포먼스
Spotify duration_ms 저장 및 트랙리스트 UI 개선.  
Supabase max_rows 버그 발견 및 수정 — 총 평점이 진짜 수치로 표시되기 시작.

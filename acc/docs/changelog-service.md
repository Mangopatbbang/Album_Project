# 서비스 일지

아차청음사 운영 기록. 코드 변경이 아닌 서비스 전체의 흐름을 담는 문서다.  
멤버 현황, 데이터 규모, 주요 기능 출시, 방향성 변화, 기술 결정을 시간순으로 기록한다.

---

## 서비스 개요

**아차청음사(ACC)** 는 2025년 12월, 친한 친구들이 모여 함께 음반을 듣고 평가를 기록하기 위해 만든 클럽 서비스다.  
취미 활동에서 시작했지만 멤버가 늘고 기능이 쌓이면서, 지금은 **공개 서비스 전환**을 목표로 개발이 이어지고 있다.

서비스 이름 유래: 아차산 인근에서 모이던 것에서 착안. "아차청음사"는 "아차산 청음사"의 줄임말이다.

### 2026-01-14 — DB & API 전환

구글 시트 의존을 완전히 제거하고 Supabase DB를 직접 읽고 쓰는 구조로 바꿨다.

- `/api/albums` REST API 구축 (GET / POST)
- MusicBrainz API 연동 — 앨범 추가 시 트랙리스트 자동 수집

### 2026-05-28 — 레거시 코드 전면 정리

오래된 코드 구조를 정리한 정비 작업. 사용자가 보는 기능은 바뀌지 않았다.

**장르 DB 정규화**
- 구형 한국어 장르값 1,236개 → 영문 정규화값으로 일괄 전환
- 전체 1,471개 앨범이 동일한 영문 장르 체계로 통일
- 기존 변환 레이어(`koGenre()` 등) 삭제 — 10개 파일 정리

**DB 컬럼 제거**
- `albums.year` 삭제 — `release_date` 앞 4자리와 완전히 동일한 중복 컬럼. 코드 28곳 정리
- `albums.search_tags`, `albums.tags` 삭제 — DB 데이터 0건, 코드 참조 0건인 방치 컬럼

**디자인 버그 수정**
- AlbumCard 점수 바 너비 계산 `/10 → /8` (8점 만점 기준으로 수정)
- AlbumModal 장르 버튼 hover 배경색 효과 추가
- DiaryEntryCard 편집·삭제 버튼 hover 방향 수정

### 2026-06-18 — AlbumModal 안정성·성능 개선 및 모바일 깜빡임 수정

앨범 모달의 버그를 잡고 성능을 높인 작업이다.  
모달 호출부 15곳 이상이 빈 평점 배열로 모달을 열기 때문에, fetch 완료 전 내용이 한꺼번에 튀어나오는 깜빡임이 발생하고 있었다.

**레이스 컨디션 수정**

모달을 빠르게 열고 닫을 때 이미 닫힌 모달에 데이터가 뒤늦게 도착해 오류가 생기는 문제를 수정했다.

- 모달 닫힐 때 진행 중인 네트워크 요청 자동 취소
- 기존 3개 분리 요청(개인 메모 / 평점 이력 / 찜 여부) → 병렬 요청으로 통합
- 5초 지연 삭제 타이머: 모달이 먼저 닫혀도 setState가 실행되던 문제 수정

**낙관적 업데이트 롤백**

좋아요·소감 좋아요 버튼을 눌렀을 때 서버 오류가 나면 UI가 틀린 상태로 굳는 문제를 수정했다.
API 실패 시 버튼 상태를 자동으로 되돌린다.

**평점 즉시 반영**

평점 저장 후 멤버 평점 목록과 평균 점수가 서버 응답을 기다리지 않고 즉시 업데이트된다.  
CDN 30초 캐시로 인해 저장 후에도 구버전 데이터가 돌아오던 문제도 함께 수정했다.

**렌더링 성능 최적화**

멤버 정렬, 논란 지수, 인기 트랙 계산 등 무거운 연산이 매 렌더마다 반복 실행되던 것을 필요할 때만 재계산하도록 변경했다.  
커버 이미지가 이미 캐시된 경우 화면에 첫 표시될 때 흰 프레임이 잠깐 보이던 현상도 제거했다.

**알림 컨텍스트 최적화**

알림 데이터가 바뀌지 않았는데도 알림을 구독하는 모든 컴포넌트가 불필요하게 다시 그려지던 문제 수정.  
프로필의 일부 필드만 바뀌어도 알림을 다시 불러오던 문제도 함께 수정.

**모바일 깜빡임 개선**

모달이 열리고 약 0.5~1초 후 트랙리스트 / 발매일 / 지역 배지가 한꺼번에 등장하면서 레이아웃이 급격히 팽창하는 깜빡임을 개선했다.

- 트랙리스트 자리에 shimmer 스켈레톤을 먼저 표시 — 데이터 로드 후 높이 변화 없이 내용만 교체됨
- 트랙리스트 / 발매일 / 지역 배지 등장 시 부드러운 fade-in(0.2s) 적용
